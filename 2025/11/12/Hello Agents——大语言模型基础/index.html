Hello Agents——大语言模型基础

第一题

自然语言处理中，语言模型经历了从统计到神经网络的模型演进。

  • 请使用本章提供的迷你语料库（datawhale agent learns, datawhale agent works），计算句子 agent works 在Bigram模型下的概率
  • N-gram模型的核心假设是马尔可夫假设。请解释这个假设的含义，以及N-gram模型存在哪些根本性局限？
  • 神经网络语言模型（RNN/LSTM）和Transformer分别是如何克服N-gram模型局限的？它们各自的优势是什么？

答：(1) 第一个词 agent 出现的概率为
$$
P(agent)=\frac{2}{6}\approx0.33
$$
计算条件概率$P(works|agent)=\frac{1}{2}=0.5$

句子 agent works 在Bigram模型下的概率为$P(agent)*P(works|agent)=0.165$

(2) 马尔科夫假设的含义是指，在随机过程中(个人觉得这个前提很重要)，它未来的状态仅由当前状态决定，与以前的状态无关。例如，明天下不下雨只和今天刮不刮风有关，与以前的天气情况有关(隐约感觉这个例子不严谨，但暂时还没想到更好的)。

N-gram只考虑前面N-1个词对下一个词出现概率的影响，有着很明显的缺陷，再早的词语与当前词的关联就被忽略了，还有就是，某个词的出现往往不是与前面的词有关，而是与后面的词有关，就像”因为下雨，所以__”，如果没有后面的词来提示，这里可能填停课，也可能填休息。此外，有的词的出现不是依靠前面的词预测的，前面的词根本就没有提示信息，只能靠先验知识，比如”东京是xx的首都”。

(3) RNN设计了隐藏状态向量作为记忆模块，每一个神经网络节点都会接收当前的输入以及上一个节点的记忆，产生新的记忆。优势在于具备了一定的记忆能力

LSTM引入了细胞状态和门控机制，有选择的记忆和遗忘信息。优势在于解决了RNN的短期记忆问题，能够具备一定的长期记忆能力

第二题

Transformer架构[4]是现代大语言模型的基础。其中：

提示：可以结合本章3.1.2节的代码实现来辅助理解

  • 自注意力机制（Self-Attention）的核心思想是什么？
  • 为什么Transformer能够并行处理序列，而RNN必须串行处理？位置编码（Positional Encoding）在其中起什么作用？
  • Decoder-Only架构与完整的Encoder-Decoder架构有什么区别？为什么现在主流的大语言模型都采用Decoder-Only架构？

答：(1) 通过计算一个词在句子中与其他所有词的相关性(也可以叫注意力分数)，再和这个词的值元素本身相乘求和。核心思想就是捕捉一个句子中每个词和其他词的内在联系吧。

(2) 因为Transformer采用的是自注意力机制，Encoder和Decoder都支持并行化，在Encoder部分，输出与之前的输入都有关系，而不是只依赖前一个，因此可以并行化处理所有自注意力机制。在Decoder部分(只在训练阶段)，引入了”teacher force”，**在训练过程中，模型的每一步输入都由真实的、来自训练数据的标签（即 “教师信号”）来提供，而不是使用模型自身上一步的预测输出。**每一个时刻的输入根本不关心上一时刻的输出，而是所有正确的label。正是这种不依赖前一刻信息的方式，关注全局，使得Transformer与RNN不同，能够并行化。

而RNN的第t个词的输入必须等待前面第t-1的隐藏状态的输出，才能结合产生第t个词的隐藏状态的输出，RNN天然的是一个时序结构，只能等前一个的进行完，才能开始现在的。

位置编码（Positional Encoding）让模型记住词元之间的时序信息(或者说顺序关系)

(3) 区别在于 Decoder-Only架构舍去了编码器部分，不再关心对输入数据的理解，模型更加简单；Encoder-Decoder架构有编码器部分，能够理解复杂的输入和输出关系，但是模型会更复杂。Decoder-Only适合做生成类的任务，Encoder-Decoder适合做输出依赖输入的任务，比如翻译和总结。

因为现在的主流大模型所完成的任务基本上的主要目的都是产生内容，核心在于对”下一个词的预测”，Decoder-only的工作模式和任务完美地契合了，舍去了编码器部分，降低了训练成本，能够将海量的文本资料用于训练，让模型学习到更多的信息，此外，只有解码器使得架构更加简单，便于复杂大模型的扩展。

第三题

文本子词分词算法是大语言模型的一项关键技术，负责将文本转换为模型可处理的 token 序列。那为什么不能直接以”字符”或”单词”作为模型的输入单元？BPE（Byte Pair Encoding）算法解决了什么问题？

文本序列转换为数字序列的过程，就叫做分词 (Tokenization)

答：单个”字符”是孤立的，没有任何意义的，模型还得学习将字符组合成单词，这会降低模型效率；”单词”的数量太过庞大，难以管理，而且无法处理新词和组合词。

BPE算法解决了单词数量庞大带来的梯度爆炸问题，同时它又能学会将字符组合成单词，而不用让模型学习，遇到新词和组合词，可以通过拆解、拼接来学习，扩充词表。

第五题

模型幻觉（Hallucination）[11]是大语言模型当前存在的关键局限性之一。本章介绍了缓解幻觉的方法（如检索增强生成、多步推理、外部工具调用）

  • 请选择其中一种，说明其工作原理和适用场景
  • 调研前沿的研究和论文，是否还有其他的缓解模型幻觉的方法，他们又有哪些改进和优势？

答：(1)RAG(检索增强生成)，这里以Advanced RAG为例，原理是通过检索知识库中的特定知识，结合检索的结果增强提示词，LLM生成基于增强提示词的结果。

适用场景：

  • 特定的领域知识，如金融，法律，医疗
  • 考虑适用时效性的知识的场景

(2)《Chain-of-Verification Reduces Hallucination in Large Language Models》，这篇论文提到了一个方法叫CoVe的方法，说简单点就是，你问大模型一个问题，大模型先生成一个初始的草稿回答，它自己呢对它的回答提出一些验证问题，看自己能不能独立地分别回答出这些问题，根据验证的结果，综合返回一个回答。

举个例子，问大模型唐朝有哪些皇帝啊？

大模型先给一个初始的回答

大模型会生成一系列事实检查的验证问题，

模型独立回答问题

根据验证结果，回答

这个方法主要在模型的初始回答，查询类的问题回答方面做了改进，在长文本生成，列表式回答方面有优势。

第六题

假设你要设计一个论文辅助阅读智能体，它能够帮助研究人员快速阅读并理解学术论文，包括：总结论文研究的核心内容、回答关于论文的问题、提取关键信息、比较多篇不同论文的观点等。请回答：

  • 你会选择哪个模型作为智能体设计时的基座模型？选择时需要考虑哪些因素？
  • 如何设计提示词来引导模型更好地理解学术论文？学术论文通常很长，可能超过模型的上下文窗口限制，你会如何解决这个问题？
  • 学术研究是严谨的，这意味着我们需要确保智能体生成的信息是准确客观忠于原文的。你认为系统中加入哪些设计能够更好的实现这一需求？

答：(1) 底座模型考虑采用国内的开源模型，如deepseek,Qwen。考虑到的因素有性能方面(deepseek，Qwen在LLM排行榜上都名列前茅)、安全隐私方面(用户看论文的请求数据也是隐私，开源模型可以部署到本地上，能很好的保护用户隐私)、成本方面(论文这种长文本会消耗大量的token，部署开源模型更有利于节省成本)

(2) 提示词会考虑使用思维链，让模型一步步思考，展示它的思考过程有利于验证它对论文的理解是否正确，还可以加入一些人类理解论文的方法。先考虑文本分割，模型按论文的目录，逐步理解，每理解一个部分，模型就可以及时总结，压缩文本信息，既保证不会超出限制，又能全面的理解。还有一个方法，就是从用户入手，说明学术论文的上传页数限制，让用户做一定的文本分割处理，这种处理可能比LLM自己处理更好。

(3) 加入RAG模块，RAG模块引入了知识库，将阅读的论文放入知识库检索，能大大提高智能体输出的客观性，智能体总结出的核心内容往往是分点作答的，它必须给出是从文章的哪个地方总结出来的，回答有关于论文的问题时，也得有理有据，以论文的用词，观点优先。
