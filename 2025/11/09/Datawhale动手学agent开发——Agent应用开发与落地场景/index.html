Datawhale动手学agent开发——Agent应用开发与落地场景

《Agent应用开发与落地全景》课题调研

​ 调研报告采用Gemini推出的DeepResearch工具生成，报告的篇幅很长，在此不便展示，结合鲁力老师的分享，将要点总结如下。

  • 首先是关于agent是什么，对于这个问题并没有明确的答案，目前可以用agent = LLM + 记忆 + 工具 + 规划 来说明其特性。个人倾向于理解为跳出文字交互这个限制，能直接自行调用工具的AI。就比如，你想用Blender做一个人物模型，现有的AI大多只能给出一个教程，教你怎么做，但agent应该能做到直接对Blender操作，将结果呈现出来即可。
  • 其次是agent的分类，一种是互动式前端智能体，如AI客服和创意助手；另一类则是自主后台智能体，与人类互动有限。但这种解决单一问题的智能体面对复杂的问题往往效果不是很好，这就需要多智能体系统（MAS），将任务拆分为多个代理来做，由此也引出了agentic system架构中的重要一类：工作流。
  • 为了⾼效构建 AI 智能体，开发框架已成为不可或缺的工具。这些框架通过提供预设组件、抽象概念和最佳实践，大大减少了开发所需的时间和精力。这里给出几个框架并对比
框架 难度 定位
langchain 难 通用应用构建与工作流编排
LlamaIndex 容易 数据连接与检索增强
AutoGen 适中 多智能体对话与协作编排
  • 对于企业来讲，agent开发的价值在于“降本，增效，赋能”。进行agent开发的原因主要是解决企业的核心痛点：复杂、重复、低效的工作流。 agent通过提供端到端的流程协调和自动化能力，与企业现有系统结合，形成跨部门、跨系统的自动化工作流。
  • 智能体的应用正在从简单的客户服务问答（前端）向复杂的后台工作流自动化和全新的用户体验模式延伸。
  • 对于agent开发，不仅要关注它的技术，也要关注现有技术条件下面临的阻碍。首当其冲的就是著名的AI“幻觉”问题，AI极有可能编造不存在的事实或提供错误结论，其次是缺乏长期记忆，容易伤害用户体验，最后则是长输入和API调用造成的token付费成本居高不下。
  • 从未来趋势上来讲，agent将会减少人类干预，从虚拟环境走向实体设备，同时不再只依赖人类训练数据，而是边做边学，自主进化。
  • 对于企业来讲，开发agent要做好三点，一是从小处着手，先验证价值再投入，可以先从内部任务自动化开始，二是构建“人机合作”的组织架构，帮助员工熟悉agent的使用，三是制定出AI治理和风险控制策略。

Agent应用金融领域产品盘点和理解

  • TradingAgents：这是一个由加利福尼亚大学洛杉矶分校，麻省理工学院和Tauric Research研究出的股票交易框架。这是一个典型的多智能体系统案例，它的特点在于通过 ReAct 提示框架，实现推理与行动的完美结合，每个智能体都能根据环境状态进行上下文相关的决策，实现多智能体的协作。采用结构化通信协议，明确每个代理的输入输出，代理仅查询和处理所需信息，并以格式化报告返回，确保交互清晰、责任明确。并且有着高度的可解释性，交易者可以轻松理解和调试系统。
  • 金灵AI：这是一个专业的金融深度投研AI Agent，基于豆包pro和Deepseek满血版模型，结合强大的数据分析、资讯搜索和研报生成能力，为用户提供专业的金融分析服务。金灵AI具备丰富的金融专业知识，能熟练运用金融工具，通过代码生成和数据计算，快速生成精准的分析结果，支持绘制金融图表，帮助用户更直观地理解市场动态。金灵AI接入了独家的实时金融数据库，涵盖财报、研报和市场热点等多维度数据，为用户打造全面、深入的投研体验。金灵AI通过多种Agent协同工作，如Search_Agent实时搜索、Code_Agent编写代码等，让分析流程高效、精准。
  • FinGenius：这是全球首个A股AI金融博弈智能体应用。FinGenius基于多Agent博弈架构，模拟市场参与者行为，用16个超级智能体（如舆情、游资、风控等）分工协作，快速生成精准的金融分析报告。FinGenius 引入博弈论优化决策，结合“年轮记忆规则算法”记录用户投资习惯，提供个性化分析。FinGenius 支持用AI重塑金融分析，提升决策效率，为投资者和金融机构提供创新解决方案。
  • Bobby：这是新加坡金融科技公司RockFlow推出的全球首个金融AI投资伙伴。通过深度学习与自然语言交互技术，能理解用户的投资需求提供个性化服务。用户可以用自然语言与Bobby交流，表达投资想法，生成相应的投资策略，包括复杂的期权策略。Bobby能实时监控市场动态，结合多维度信息为用户提供交易机会，根据用户风险偏好进行持仓管理和风险控制，可以设置智能提醒和自动下单。

Agent应用的核心技术栈和学习方法

核心技术栈

大语言模型（LLM）， 是 Agent 的 “语言中枢”，负责处理自然语言指令、生成逻辑决策、理解文本信息，是绝大多数 Agent 的核心驱动组件。代表模型：GPT系列、Claude 3、Llama 3、Qwen（通义千问）、文心一言。要学习的技术有：

  • 上下文窗口扩展（如 GPT-4o 的 128k tokens）：支持 Agent 处理长文本任务（如分析完整报告、记忆多轮对话历史）；
  • Function Calling：LLM 通过输出结构化指令（如 API 参数），调用外部工具（如计算器、搜索引擎），突破自身能力边界；
  • 微调技术（SFT/Finetuning）：通过领域数据微调，让 Agent 适配特定场景（如医疗 Agent 用医学数据微调、金融 Agent 用财报数据微调）。

规划。所谓规划，就是将用户的复杂任务拆解为 “可执行的子任务序列”（如 “确定时间地点→联系供应商→制定流程→通知员工”），要学习的技术为利用 LLM 的逻辑推理能力，直接生成子任务序列（如 Prompt 提示 “请将‘策划年会’拆解为 5 个步骤，按时间顺序排列”）。

决策，针对每个子任务，Agent 需决策 “采取什么动作”（如子任务 “获取天气信息”，决策 “调用天气 API” 还是 “搜索网页天气数据”），要学习的技术为强化学习，通过 “奖励机制” 优化决策 —— 如 Agent 选择 “调用 API” 时响应更快（奖励 + 1），选择 “网页搜索” 时耗时久（奖励 - 1），长期训练后优先选择最优动作。

工具调用，Agent 自身无法直接访问外部数据或执行复杂操作，需通过调用工具实现，要学习的技术有：

  • API 类工具：调用第三方服务 API（如天气 API、支付 API、地图 API）；
  • agent 框架：为简化工具调用逻辑，常用框架（如 LangChain、LlamaIndex、AutoGen）提供 “工具注册 - 调用 - 结果解析” 的标准化接口，Agent 只需配置工具名称和参数格式，即可快速集成。
  • MCP：一种连接 AI 助手与外部系统的开放标准，使模型能够获取上下文信息，为 Agent 提供与外部系统交互的标准化接口。Agent 可以通过 MCP 调用工具、获取数据，从而扩展其行动能力。

记忆与知识库，Agent 需 “记住” 历史交互信息和领域知识，才能实现个性化、精准化决策，要学习的技术为RAG ：结合信息检索与生成模型，通过从外部知识库中提取信息，增强生成内容的准确性，可作为 Agent 的知识组件，为决策提供事实基础

学习方法

  1. 开发语言选择Python：掌握 Python 基础语法（函数、类、模块）、数据结构（列表、字典、集合）、常用库（requests（工具调用 API）、pandas（记忆数据处理）、json（数据序列化）、asyncio（异步工具调用））。

  2. Agent 的核心是 “大模型驱动”，需理解大模型的基本原理，才能合理选择模型、调优 Prompt，可以看李沐《动手学深度学习》（Transformer 章节），或 OpenAI 的《GPT-4 Technical Report》进行理论学习。

  3. 参考开源项目，逆向学习，Agent 领域有大量优质开源项目，可通过 “跑通代码→修改功能→自定义开发” 的流程快速提升。

  4. 学习agent 最重要的是练习，可以从小的demo开始，再从单智能体系统到多智能体系统。
